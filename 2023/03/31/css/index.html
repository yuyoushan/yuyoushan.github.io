2.BFC

块级格式化上下文(块级元素),是页面中一块渲染区域,并且有一套属于自己的渲染规则.

布局规则:

  1. 内部盒子会在垂直方向上一个接一个的排列
  2. 内部盒子的元素布局不会影响到外部元素
  3. 同一个BFC的两个相邻的盒子的margin会发生重叠,与方向无关
  4. 计算BFC的高度时,浮动子元素也参与计算

触发条件(产生一个BFC):

  1. 根元素,HTML元素
  2. 浮动元素,float为left,right
  3. overflow(元素溢出):值为auto(内容溢出,产生滚动条),scroll(产生滚动条),hidden(溢出隐藏)
  4. display:inline-block,inline-cell,table
  5. position(定位):值为absolute(绝对定位)或fixed(固定定位)

盒子模型

盒子组成:content(内容),padding(内边距),border(边框),margin(外边距)

  1. 标准盒子模型:content+border+padding
  2. IE盒子模型:content(包含了padding和border)

flexbox弹性盒子布局模型

  1. 是一种布局模型,给父元素添加display:flex,子元素可以自动的挤压或拉伸,组成的部分有弹性容器,弹性盒子,主轴,侧轴
  2. 应用场景:可以实现元素水平垂直居中以及左右固定,中间自适应,移动端通常使用flex布局

flex-direction:决定主轴的方向

flex-wrap:决定容器内项目是否换行

justify-content:决定了项目在主轴上的对齐方式(弹性容器)

align-items:决定项目侧轴的对齐方式(弹性容器)

flex:伸缩比,属性值为数字,只占用父盒子剩余尺寸(弹性盒子)

Css选择器

  1. 基础选择器: id,class,标签
  2. 复合选择器: 后代,子,并集,交集
  3. 其他:伪类,伪元素,通配符,属性

优先级:

​ 继承<通配符<标签<类<id<行内<!important

px/em/rem/vh/vw

  1. px:绝对单位,页面按照精确像素展示
  2. em:相对单位,默认为浏览器的默认字体尺寸(1em=16px),会继承父级元素的字体大小
  3. rem:相对单位,相对根节点html1的字体大小来计算
  4. vh、vw:根据页面视口的宽高,分成100等分

响应式布局

  1. 基本原理:通过媒体查询检查不同的设备屏幕尺寸,页面头部必有有meta标签并且声明viewport
  2. 通过百分比,vw/vh,rem,或者ui组件设置响应式布局

文本溢出的省略样式

1.单行文本溢出换省略号

2.多行文本溢出换省略号

重绘与回流

1.重绘: 页面节点的样式修改,不影响它在文档流中的位置和布局

2.回流: 元素的尺寸,布局发生改变时,浏览器会重新渲染部分或全部文档的过程

重绘不一定引起回流,但回流一定会引起重绘
